========================================
PROJETO SMARTCLINIC
FASE 1 – SISTEMA CRUD
========================================

1. DEFINIÇÃO DO TEMA
---------------------
Sistema de gestão de clínica com controle de pacientes e estoque de produtos/serviços.

2. DEFINIÇÃO DA ENTIDADE PRINCIPAL
-----------------------------------
Entidades escolhidas para o CRUD inicial:

- Cliente (Paciente)
- Produto/Serviço

3. LEVANTAMENTO DOS ATRIBUTOS
------------------------------

3.1 Cliente
- id_cliente (PK)
- nome
- cpf
- telefone
- data_nascimento
- cidade
- torce_flamengo (boolean)
- assiste_one_piece (boolean)

3.2 Produto/Serviço
- id_produto (PK)
- nome
- categoria
- preco
- tipo_item (PRODUTO ou SERVICO)
- quantidade_estoque
- fabricado_em_mari (boolean)

4. MODELAGEM UML (CLASSES)
---------------------------
Criar diagrama de classes contendo:

Classe Cliente
Classe Produto
Classe CrudManager (classe responsável por gerenciar operações CRUD)

Métodos obrigatórios:
- inserir()
- alterar()
- pesquisarPorNome()
- remover()
- listarTodos()
- exibirUm()

5. CRIAÇÃO DO BANCO DE DADOS
------------------------------
- Criar o banco de dados
- Criar tabelas Cliente e Produto
- Definir:
  - Chave primária
  - Chave única (cpf)
  - Campos NOT NULL
  - Tipos corretos (VARCHAR, DATE, DECIMAL, BOOLEAN)

6. IMPLEMENTAÇÃO DO CRUD
-------------------------
Para cada entidade implementar:

1. Inserir registro
2. Alterar registro
3. Pesquisar por nome
4. Remover registro
5. Listar todos
6. Exibir um registro específico

7. REGRAS BÁSICAS
------------------
- CPF deve ser único
- Produto não pode ter preço negativo
- Quantidade em estoque não pode ser negativa

8. RELATÓRIO SIMPLES
---------------------
Gerar relatório contendo:

- Quantidade total de clientes cadastrados
- Quantidade total de produtos
- Valor total em estoque (soma de preço * quantidade)

9. TESTES
---------
- Inserir dados de exemplo
- Testar todas as operações
- Validar integridade dos dados

10. ORGANIZAÇÃO DO PROJETO
---------------------------
Estrutura sugerida:

/database
    schema.sql
/src
    cliente.py
    produto.py
    crud_manager.py
main.py
README.md
